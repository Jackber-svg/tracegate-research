# TraceGate Research Framework - Release 1.0

## Positioning

TraceGate Research is a file-grounded governance protocol for long-running research agent projects. It prevents agents from silently drifting from strict research contracts into convenient completion mode by forcing state, evidence, artifacts, decisions, parameters, adapter outputs, and checkpoints through explicit gates.

The framework is designed for research workflows where a wrong but complete-looking answer is worse than a blocked task: scientific modeling, computational simulation, research-code generation, data-analysis pipelines, literature-derived parameterization, and any workflow that must survive handoff between agents or sessions.

The core rule is simple:

```text
Do not trust memory, summaries, or good-looking outputs.
Trust only files, hashes, gates, manifests, decisions, and reproducible artifacts.
```

Gate-Driven Agent does not prove that a result is true. It makes drift, missing evidence, proxy use, source-copy errors, equation simplification, extension leakage, and self-audit leniency visible early enough to stop them.

## Six Questions

Every long-running research agent project must answer six questions from files, not from conversation history:

```text
Where are we now?          STATE.json
What counts as correct?    CONTRACT.yaml and TASK_CONTRACT.yaml
Where do values come from? PARAMETER_REGISTRY.json and SOURCE_MANIFEST.json
Did it pass inspection?    GATE_REPORTS/
Which artifacts exist?     ARTIFACT_MANIFEST.json
Who approved exceptions?   DECISIONS.jsonl
```

If a new agent cannot answer these questions from the project directory alone, the project is not handoff-safe.

## 1. Project Layout

```text
project/
  STATE.json                       required
  CONTRACT.yaml                    required
  TASK_CONTRACT.yaml               optional split contract, same hash domain
  ARTIFACT_MANIFEST.json           required
  DECISIONS.jsonl                  required in full profile
  GATE_REPORTS/                    required

  PARAMETER_REGISTRY.json          full profile or parameterized projects
  SOURCE_MANIFEST.json             when source_lock_gate is declared
  EQUATION_MANIFEST.json           when equation_form_gate is declared
  EXTENSION_KEYWORD_MANIFEST.json  when extension_residual_gate is declared
  UNIT_REGISTRY.json               when unit normalization is used

  ADAPTER.yaml                     when an external tool/model/runtime is inspected
  MODEL_STATE.json                 generated by adapter
  runtime_expression_dump.json     generated when adapter supports extract_expressions

  schemas/                         machine-readable schemas
  runner/                          gate runner scripts
  references/                      explanatory docs and examples
```

Conditional manifest files are required only when their corresponding gate is declared in `CONTRACT.yaml` or required by the current mode. If a gate is declared but its required manifest is missing, the result is `BLOCK`. If the gate is not declared and the manifest is absent, the result is `SKIPPED_NOT_CONFIGURED`.

## 2. Closed Enums

Use closed enums. Free-text state fields are protocol leaks.

```text
ARTIFACT_TYPE
model | code | data | config | document | result | gate_report | adapter |
adapter_output | api_request | api_response | database_snapshot |
decision_log | manifest | contract | registry | checkpoint |
approval_record | model_state | source_manifest | equation_manifest |
extension_keyword_manifest | task_contract | unit_registry

ARTIFACT_STATUS
current | verified | staging | evidence | archived | superseded |
rejected | failed | external | unknown

SEVERITY
INFO | WARN | BLOCK | CRITICAL

DECISION_STATUS
OPEN | ACCEPTED | REJECTED | MITIGATED | SUPERSEDED

DECISION_KIND
warning_resolution | parameter_acceptance | threshold_change |
mode_transition | rollback | human_override | action_change |
adapter_missing | external_anchor_exception | source_incomplete_resolution |
external_audit_resolution

SOURCE_STATUS
SOURCE_VERIFIED | SOURCE_RANGE | SOURCE_DERIVED |
SOURCE_INCOMPLETE | SOURCE_MISSING | SOURCE_REJECTED

EVIDENCE_GRADE
A  Same system, same condition, DOI or stable source available.
B  Same system, different condition, or close analogue with source.
C  Cross-system transfer, derived, or weak single source.
D  Bounded estimate, wide sweep, figure digitization, or incomplete source.
E  Missing.
F  Known wrong.
NA Not applicable.

ADAPTER_CAPABILITY
open_artifact | list_object_tree | extract_parameters | extract_materials |
extract_boundaries | extract_studies | extract_results | extract_messages |
extract_expressions | export_model_state | run_solver | run_postprocess |
read_api_response | read_database_snapshot
```

`extract_expressions` is the capability that allows an adapter to export `runtime_expression_dump.json` for `equation_form_gate`. Without it, `equation_form_gate` may be skipped only as `SKIPPED_NOT_APPLICABLE`, never as `PASS`.

## 3. Modes and Gate Profiles

```text
DISCOVERY
Unknowns allowed if explicitly marked GAP, HYPOTHESIS, PROXY, or SOURCE_INCOMPLETE.
Final claims and baseline promotion are forbidden.

STAGING
Building and testing. BLOCK stops work. WARN may continue, but cannot promote baseline.

VALIDATION
Evidence closure required. Domain gates and source locks must run.

BASELINE
Publishable or handoff-authoritative state. No OPEN decisions.
```

| mode | lite required gates | full additional gates |
|---|---|---|
| DISCOVERY | contract_load_gate, schema_gate, manifest_gate, action_gate | decision_sync_gate, artifact_existence_gate |
| STAGING | contract_load_gate, schema_gate, hash_gate, manifest_gate, action_gate | registry_gate, adapter_export_gate if adapter capable, unit_normalization_gate if units exist |
| VALIDATION | contract_load_gate, schema_gate, hash_gate, manifest_gate, action_gate | adapter gates, registry_closure_gate, source_lock_gate when declared, decision_gate, decision_sync_gate, external_audit_gate when declared, non-optional domain gates |
| BASELINE | contract_load_gate, schema_gate, hash_gate, manifest_gate, action_gate | all required adapter gates, registry_closure_gate, source_lock_gate when declared, extension_residual_gate when declared, equation_form_gate when declared, decision_gate, decision_sync_gate, external_audit_gate when declared, non-optional domain gates, zero OPEN decisions |

Domain gates are required by default. They can be optional only if the rule has `optional: true` and a non-empty `optional_reason`.

## 4. STATE.json

`STATE.json` is the current authority pointer. It should be small, hash-bound, and readable at startup.

```json
{
  "project": "my_project",
  "mode": "STAGING",
  "profile": "full",
  "contract": {
    "files": {
      "CONTRACT.yaml": "sha256:abc123",
      "TASK_CONTRACT.yaml": "sha256:def456"
    },
    "primary": "CONTRACT.yaml"
  },
  "current_artifact": {
    "artifact_id": "ART-001",
    "type": "model",
    "path": "model.mph",
    "hash": "sha256:...",
    "adapter_id": "comsol"
  },
  "last_checkpoint": {
    "checkpoint_id": "CHK-003",
    "timestamp": "2026-06-23T14:32:00Z",
    "gate_report": "GATE_REPORTS/gate_report_003.json",
    "all_required_gates_pass": true
  },
  "artifact_manifest_hash": "sha256:...",
  "external_anchor": {
    "type": "git_commit",
    "value": "abc123"
  },
  "next_allowed_actions": [],
  "blocked_actions": [],
  "open_decisions": []
}
```

The `contract.files` map is important. Long-context constraints must be loaded from contract files and verified by hash before any other gate runs.

## 5. CONTRACT.yaml

`CONTRACT.yaml` defines what the project is allowed to do and what counts as passing. It is not a prompt. It is a machine-checkable task contract.

```yaml
version: "1.0"
project: "my_project"
includes:
  - TASK_CONTRACT.yaml

rules:
  - id: CORE01
    layer: core
    category: contract_load_gate
    check_type: contract_load_check
    input_file: CONTRACT.yaml
    params:
      contract_files: ["CONTRACT.yaml", "TASK_CONTRACT.yaml"]
      expected_hash_source: STATE.json:$.contract.files
      require_readable: true
      block_on_hash_mismatch: true
    fail_level: BLOCK
    output_artifact: GATE_REPORTS/contract_load.json

  - id: SRC01
    layer: core
    category: source_lock_gate
    check_type: source_original_value_cross_check
    input_file: PARAMETER_REGISTRY.json
    params:
      source_manifest_file: SOURCE_MANIFEST.json
      join_key: parameter
      require_source_anchor: true
      require_conversion_record: true
      tolerance_policy: from_registry_comparison_field
    fail_level: BLOCK
    output_artifact: GATE_REPORTS/source_lock.json

  - id: EXT01
    layer: core
    category: extension_residual_gate
    check_type: extension_residual_scan
    input_file: EXTENSION_KEYWORD_MANIFEST.json
    params:
      runtime_artifacts: ["MODEL_STATE.json", "runtime_expression_dump.json", "src/"]
      mode: CORE
      fail_on_forbidden_token: true
    fail_level: BLOCK
    output_artifact: GATE_REPORTS/extension_residual.json

  - id: EQN01
    layer: core
    category: equation_form_gate
    check_type: equation_form_check
    input_file: EQUATION_MANIFEST.json
    params:
      runtime_expression_dump: runtime_expression_dump.json
      compare:
        - term_count
        - variable_list
        - nonlinearity_class
        - sign_vector
        - required_terms
        - forbidden_terms
    fail_level: BLOCK
    output_artifact: GATE_REPORTS/equation_form.json
    skip_if: runtime_expression_dump_not_available
```

| equation_form_gate | equation_form_check, cross_artifact_match | declared or EQUATION_MANIFEST exists |
| artifact_existence_gate | artifact_existence_check, json_array_scan, hash_verify | full profile |
| unit_normalization_gate | unit_normalization_check, unit_conversion_check | units present |
| adapter_capability_gate | adapter_capability_check | adapter exists |
| adapter_export_gate | adapter_export, schema_validate | adapter capable |
| adapter_output_binding_gate | cross_artifact_match | adapter exports |

## 8. Source Locking

Registration is not enough. A parameter can be registered and still be wrong because of a copied digit, missed sign, axis multiplier, or unit conversion error.

`SOURCE_MANIFEST.json` records what the source actually says and how it becomes the encoded model value.

```json
{
  "version": "1.0",
  "sources": [{
    "source_id": "SRC-EXAMPLE-FIG7",
    "artifact_id": "ART-SOURCE-PDF-001",
    "locator": "Fig.7a",
    "source_status": "SOURCE_INCOMPLETE",
    "source_incomplete_reason": "coefficients appear only as a plotted curve, not as a table",
    "extraction_method": "digitization",
    "extraction_uncertainty": {"type": "relative_error", "value": 0.03},
    "independent_qa_artifact": "ART-WPD-QA-001",
    "allowed_claim_level": "FIGURE_DIGITIZED"
  }],
  "values": [{
    "parameter": "D_Li_CF",
    "source_id": "SRC-EXAMPLE-FIG7",
    "original_value": {"type": "range", "data": {"min": 0.0, "max": 4.0}, "unit": "1e-12 m2/s"},
    "encoded_value": {"type": "range", "data": {"min": 0.0, "max": 4.0e-12}, "unit": "m2/s"},
    "conversion_factor": 1e-12,
    "conversion_note": "axis multiplier 1e-12 applied"
  }]
}
```

Rules:

```text
For every baseline_allowed parameter with source_anchor:
  registry.value must match source_manifest.encoded_value within PARAMETER_REGISTRY.rows[*].comparison.tolerance.
  registry.source_anchor.original_value must match source_manifest.original_value.
  If units differ, conversion_factor and conversion_note are required.
  If source_status is SOURCE_INCOMPLETE, source_decision_id is required.
```

## 9. Source-Incomplete Decisions

`SOURCE_INCOMPLETE` is not the same as `SOURCE_MISSING`. It means the source contains the concept, equation, figure, or derivation, but does not provide a directly encodable value or coefficient set.

Example decision row:

```json
{
  "decision_id": "D-SRC-001",
  "decision_kind": "source_incomplete_resolution",
  "status": "ACCEPTED",
  "severity": "WARN",
  "applies_to": {"source_id": "SRC-EXAMPLE-FIG7", "parameter": "D_Li_CF"},
  "source_status": "SOURCE_INCOMPLETE",
  "source_incomplete_reason": "Coefficients are available only in plotted form.",
  "extraction_method": "digitization",
  "extraction_uncertainty": {"type": "relative_error", "value": 0.03},
  "independent_qa_artifact": "ART-WPD-QA-001",
  "allowed_claim_level": "FIGURE_DIGITIZED",
  "approved_by_type": "human",
  "approved_by_id": "user-local",
  "approval_source": "manual_entry",
  "approval_artifact_id": "ART-APPROVAL-001",
  "timestamp": "2026-06-23T15:00:00Z",
  "rationale": "Digitized values are accepted for screening, but cannot be claimed as directly tabulated source data."
}
```

Allowed claim levels:

```text
DIRECT_SOURCE_VALUE
LITERATURE_RANGE
FIGURE_DIGITIZED
EXTERNAL_SOURCE_PROXY
DERIVED_FROM_FORMULA
INFERRED
NOT_FOR_BASELINE
```

Rules:

```text
SOURCE_INCOMPLETE cannot silently become SOURCE_VERIFIED.
FIGURE_DIGITIZED requires independent_qa_artifact and extraction_uncertainty.
EXTERNAL_SOURCE_PROXY requires a separate source_id.
INFERRED cannot enter BASELINE unless explicitly accepted by human or external auditor.
```

## 10. PARAMETER_REGISTRY.json

The registry separates source kind, evidence strength, source completeness, baseline permission, implementation binding, and hash lineage.

```json
{
  "version": "1.0",
  "rows": [{
    "parameter": "D_Li_CF",
    "value": {"type": "number", "data": 3.6e-12, "unit": "m2/s"},
    "source_class": "literature_range",
    "source_anchor": {
      "source_id": "SRC-EXAMPLE-FIG7",
      "locator": "Fig.7a",
      "original_value": {"type": "number", "data": 3.6, "unit": "1e-12 m2/s"},
      "conversion_factor": 1e-12
    },
    "source_status": "SOURCE_INCOMPLETE",
    "source_decision_id": "D-SRC-001",
    "evidence_grade": "D",
    "baseline_allowed": true,
    "requires_sweep": true,
    "requires_human_acceptance": false,
    "sweep_status": "PASS",
    "sweep_report_id": "SWEEP-001",
    "implementation_source": "MODEL_STATE.json:$.parameters[?(@.name=='D_Li_CF')].value",
    "implementation_value": {"type": "number", "data": 3.6e-12, "unit": "m2/s"},
    "comparison": {"operator": "numeric_close", "tolerance": 1e-14, "unit_policy": "normalize"},
    "row_hash": "sha256:...",
    "source_hash": "sha256:...",
    "implementation_hash": "sha256:..."
  }]
}
```

Three classification axes must not be collapsed:

| axis | answers |
|---|---|
| `source_class` | What kind of source is this? |
| `evidence_grade` | How strong is the evidence? |
| `source_status` | Is the source complete enough to encode directly? |

Baseline permission requires:

```text
source_class not in [diagnostic_switch, numerical_regularization]
evidence_grade in [A, B, C, NA] OR an accepted decision exists
source_status not in [SOURCE_MISSING, SOURCE_REJECTED]
SOURCE_INCOMPLETE implies source_decision_id is not null
```

## 11. Extension Residual Scan

An extension switch being off does not prove the extension is absent. Runtime artifacts must not contain active residual extension tokens in core mode.

```json
{
  "version": "1.0",
  "mode": "CORE",
  "extensions": [{
    "extension_id": "stress_diffusion",
    "switch_parameter": "enable_stress_diffusion",
    "expected_switch_value": false,
    "forbidden_tokens": [
      "stressDiffusion",
      "D_theta_cf_ext",
      "sigma_h_coupling",
      "Omega_n_ext"
    ],
    "allowed_contexts": [
      "parameter_definition",
      "documentation",
      "disabled_feature_manifest"
    ]
  }]
}
```

Rules:

```text
If an extension switch is off, forbidden tokens must not appear outside allowed_contexts.
Scan targets may be source code, model object trees, binary-derived feature dumps, database names, imports, tables, or runtime expression dumps.
Directory inputs are scanned recursively for text files; binary files are skipped; symbolic links are followed with max depth 1.
Any forbidden token in an active runtime path is BLOCK.
```

## 12. Equation Form Gate

KPI match is not equation correctness. A simplified or wrong equation can pass output gates because parameters compensate.

```json
{
  "version": "1.0",
  "equations": [{
    "equation_id": "transport_equation",
    "target_artifact": "ART-MODEL-STATE-001",
    "term_count": 4,
    "variable_list": ["c", "phi", "D", "kappa"],
    "nonlinearity_class": "coupled_nonlinear",
    "sign_vector": ["+", "-", "+", "-"],
    "required_terms": ["diffusion", "migration", "source", "boundary_flux"],
    "forbidden_terms": ["scalar_proxy_transport"]
  }]
}
```

Rules:

```text
term_count must match unless an accepted simplification decision exists.
variable_list must contain all required variables.
nonlinearity_class must match.
sign_vector must match.
required_terms must be present.
forbidden_terms must be absent.
KPI gates cannot override equation_form_gate.
```

For code projects, this maps to interface signatures, call graphs, imports, and algorithm classes. For data analysis, it maps to declared statistical formulas versus executed model calls.

## 13. External Audit

Agents are lenient toward their own work. Critical promotions should support external audit.

External audit can mean a separate model, a separate runner, a separate agent, or a manually approved audit artifact. It must be represented as a file and linked through `ARTIFACT_MANIFEST.json`.

```text
external_audit_gate passes only if:
  audit_artifact exists and is hash-registered
  findings are empty or explicitly resolved in DECISIONS.jsonl
  unresolved BLOCK/CRITICAL findings prevent VALIDATION or BASELINE
  auditor identity is recorded as human, external_model, separate_runner, or separate_agent
```

External audit is optional in DISCOVERY and STAGING, but recommended for BASELINE if the project contains scientific claims, irreversible edits, expensive simulation outputs, manuscript-facing artifacts, or public research deliverables.

## 14. Startup Protocol

Every new session must start from the project directory, not the chat transcript.

```text
1. Read STATE.json. Validate schema and closed enums.
2. Run contract_load_gate on all files listed in STATE.contract.files.
3. Read ARTIFACT_MANIFEST.json. Verify artifact_manifest_hash.
4. Determine mode and profile from STATE.
5. Read DECISIONS.jsonl. Run decision_sync_gate. Run sync hook if failed.
6. If full profile, read PARAMETER_REGISTRY.json.
7. If source_lock_gate is required, read SOURCE_MANIFEST.json and run source_original_value_cross_check.
8. If adapter exists, read ADAPTER.yaml, validate capabilities, and run adapter_export if required.
9. If equation_form_gate is required and adapter has extract_expressions, run equation_form_check after adapter_export.
10. If extension_residual_gate is required, run extension_residual_scan.
11. Run external_audit_gate if declared.
12. Run all required non-optional domain gates.
13. Answer cold-start questions and update next_allowed_actions.
```

Cold-start questions:

```text
What is the authority artifact?
What mode and profile are active?
What was the last passing gate report?
What actions are allowed and blocked?
What hard constraints were loaded from CONTRACT?
Which decisions remain OPEN?
Which parameters are SOURCE_INCOMPLETE?
Which extensions are forbidden?
Which equation forms are locked?
What is the external anchor status?
```

## 15. Checkpoint Rule

A checkpoint is valid only when:

```text
all required gates for the current mode/profile have PASS
conditional gates are either PASS or explicitly SKIPPED_NOT_CONFIGURED / SKIPPED_NOT_APPLICABLE
no BLOCK or CRITICAL finding is unresolved
BASELINE has zero OPEN decisions
ARTIFACT_MANIFEST hash matches STATE.artifact_manifest_hash
CONTRACT file hashes match STATE.contract.files
```

Failed or partial work may be archived as evidence, but must not overwrite the last verified baseline.

## 16. Release Readiness Checklist

```text
[ ] contract_load_gate runs before every other gate.
[ ] STATE.contract.files supports multiple contract file hashes.
[ ] Conditional gates BLOCK when CONTRACT declares them but manifests are missing.
[ ] Conditional gates are SKIPPED_NOT_CONFIGURED only when not declared and no manifest exists.
[ ] ADAPTER_CAPABILITY includes extract_expressions.
[ ] equation_form_gate is SKIPPED_NOT_APPLICABLE, not PASS, when extraction is unavailable.
[ ] PARAMETER_REGISTRY includes source_anchor, source_status, and source_decision_id.
[ ] source_original_value_cross_check validates original value, encoded value, and conversion.
[ ] unit_conversion_check validates dimensions, aliases, and conversion factors.
[ ] SOURCE_INCOMPLETE decision branch exists.
[ ] extension_residual_scan checks MODEL_STATE, expression dumps, and source trees.
[ ] equation_form_check runs before KPI or domain metric gates.
[ ] KPI gates cannot override equation_form_gate.
[ ] external_audit_gate can block unresolved external findings.
[ ] Cold-start test passes from project directory only.
[ ] Dummy project passes all required gates in STAGING mode.
```

## One-Sentence Summary

TraceGate Research Release 1.0 is a file-grounded, schema-bound, runner-safe protocol for long-running research agent projects: contract files prevent constraint dilution, source locks prevent evidence laundering, equation and extension gates prevent implementation drift, external audits reduce self-leniency, and checkpoints allow new agents to continue from verified research state instead of fragile chat memory.
